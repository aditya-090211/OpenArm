# Contributing

Contributions, feedback, and questions are welcome.

The project is in early development — hardware hasn't been built yet, the simulation is functional but rough in places, and a lot of the research is still evolving. That means there's room to help in a lot of directions.

---

## What's Useful Right Now

- **Simulation improvements:** Better motion planning algorithms, self-collision detection, dynamics modeling, joint limit enforcement, spline interpolation
- **IK/FK validation:** Finding edge cases or errors in the kinematic model
- **CAD feedback:** Mechanical design suggestions, manufacturability improvements, weight reduction ideas
- **Control systems research:** PID tuning strategy, encoder selection, the RL vs. PID tradeoff question (`Research/rl_vs_pid.md`)
- **Cost reduction:** The BOM is too expensive. Alternative actuator architectures, structural substitutions, cheaper encoder options
- **Documentation:** Clearer explanations, corrections, better organization

---

## How to Contribute

Open an issue or submit a pull request. For significant changes, opening an issue to discuss the approach first is helpful.

The CAD is maintained in Onshape (link in `CAD/README.md`). You can make a copy of the document and modify it independently.
